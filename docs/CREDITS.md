# Credits & Acknowledgements

AveGram exists because dozens of developers before us did the hard work.
This page is our public thank-you note.

## 🌟 Direct upstream

### [NagramXF](https://github.com/Keeperorowner/NagramXF)
The fork AveGram is based on. Maintained by **Keeperorowner**. Provides the
solid foundation, build pipeline, and most of the power-user features
inherited here.

## 🙏 Major sources we ported from

### [NagramX](https://github.com/risin42/NagramX) — by **risin42**
- Custom folder system → ported to `org.avegram.ui.folders.*`
- Helper utilities for chat management → `org.avegram.helper.*`
- Now-playing controls → `org.avegram.nowplaying.*`

### [AyuGram (4A)](https://github.com/AyuGram/AyuGram4A) — by **AyuGram team**
- Saved-deleted-messages database
- Message-log controllers
- Privacy and stealth features
- Lives under `com.radolyn.ayugram.*`

### [Cherrygram](https://github.com/arsLan4k1390/Cherrygram) — by **arsLan4k1390**
- UI refinements & component polish
- Several settings overrides

### [exteraGram](https://github.com/exteraSquad/exteraGram) — by **exteraSquad**
- Design language inspiration
- UI components → `com.exteragram.messenger.*`

## 🛠 Indirect / foundational

### [Telegram for Android](https://github.com/DrKLO/Telegram) — by **Telegram FZ-LLC** (DrKLO)
The original messenger client. Without it, none of these forks would exist.

### [Nekogram](https://github.com/NekoX-Dev/NekoX) — by **NekoX-Dev**
Pioneer of the Telegram-fork ecosystem. The `tw.nekomimi.nekogram.*`
package lineage traces back here.

## 📚 Libraries & tooling

- **OkHttp**, **Retrofit** — networking
- **ExoPlayer** — media playback
- **TgCalls / libtgvoip** — voice & video calls
- **FFmpeg**, **Opus**, **FLAC** — codecs
- **ZXing** — QR code handling
- **Material Components for Android** — base UI primitives
- **Firebase** — push notifications & crash reporting
- **Gradle**, **Android Gradle Plugin**, **CMake**, **NDK** — build system

## 💝 Community

To everyone who reports bugs, translates strings, files thoughtful issues,
sends pull requests, or simply talks about AveGram — thank you. You make
this project worth maintaining.
